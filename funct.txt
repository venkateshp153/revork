2. Login Screen
Build a React Native LoginScreen component with three vertically stacked auth sections inside a KeyboardAvoidingView (behavior: 'padding' on iOS, 'height' on Android) wrapped in a ScrollView.

SECTION 1 — Join via Link (Worker access):
- Label: "Join a Company Team"
- TextInput with placeholder "Paste join link here..." stored in local state joinLink.
- Inline row (flexDirection: 'row'): TextInput flex 1, then "Join" Button.
- Button is DISABLED (opacity 0.4, pointerEvents none) when joinLink.trim() is empty.
- On press:
  1. Parse URL using Linking or URL constructor: extract sheetId, token, companyId params.
  2. If params missing → show inline error "Invalid or expired link".
  3. Make a read-only GET request to the Google Sheet using the token.
  4. If response is non-200 or empty → show inline error "Link not recognized. Please check with your admin."
  5. If success → store raw response data in a local ref (do NOT dispatch to Redux yet).
  6. Show a Modal (WorkerVerifyModal) with three inputs: Name, Phone, Email.
  7. On "Verify" button press inside modal: check if entered phone AND email match the fetched row data (case-insensitive, trimmed). Name is informational only.
  8. If match → dispatch fetched worker profile to Redux userSlice, navigate.replace('Tabs').
  9. If no match → show red inline error inside modal: "Details don't match. Please check with your admin."
  10. Company name from sheet response → store in Redux and display top-center on Home screen.

SECTION 2 — Email / Password (Admin/Manager access):
- Label: "Manage Your Company"
- TextInput for email: validate with regex /^[^\s@]+@[^\s@]+\.[^\s@]+$/ on change.
- TextInput for password: secureTextEntry={true}, validate: minimum 8 chars, at least one number.
- Show inline red validation messages below each input as user types (not on submit).
- "Login" button: disabled until BOTH inputs pass regex validation.
- On press:
  1. Call Firebase Auth signInWithEmailAndPassword(email, password).
  2. On success → call checkIfNewUser(uid). If new → show NewUserProfileModal. If existing → dispatch profile to Redux → navigate.replace('Tabs').
  3. On Firebase error auth/user-not-found or auth/wrong-password → show "Incorrect email or password."
  4. On error auth/email-already-in-use → show "This email is already linked to another account."
  5. If user tries to sign up and email/phone already exists → Firebase returns error → show "An account with this email or phone already exists. Try a different one."

SECTION 3 — Google OAuth:
- Single "Continue with Google" button with Google logo icon.
- Use @react-native-google-signin/google-signin (configure in app entry point with your webClientId).
- On press:
  1. Call GoogleSignin.signIn() → get idToken.
  2. Credential via firebase.auth.GoogleAuthProvider.credential(idToken).
  3. Sign in with Firebase using signInWithCredential.
  4. Call checkIfNewUser(uid): if new → open NewUserProfileModal with name and email pre-filled from Google profile. If existing → dispatch to Redux → navigate.replace('Tabs').
  5. If Firebase returns error that email is linked to a different provider → show "This email is already used with a different sign-in method."

DIVIDERS:
- Between each section render a centered "(or)" text in muted color with a horizontal line on each side (use flexDirection: 'row' with View flex 1 height 1 backgroundColor '#ccc').

checkIfNewUser(uid) helper:
- Query Firestore users/{uid}. If document does not exist → return true (new). If exists → dispatch profile data to Redux → return false.

4. Google Drive & Sheets Setup Utility
Create a utility file: src/utils/createGoogleDriveStructure.js

Export an async function: createGoogleDriveStructure(companyName, accessToken)

This function runs ONCE after new Admin profile is saved. Steps:

1. FOLDER: POST to https://www.googleapis.com/drive/v3/files with Authorization: Bearer {accessToken}.
   Body: { name: `${companyName} WorkForce Data`, mimeType: 'application/vnd.google-apps.folder' }
   Store returned id as folderId.

2. SPREADSHEET: POST to https://sheets.googleapis.com/v4/spreadsheets.
   Body: { properties: { title: `${companyName} Records` }, sheets: [
     { properties: { title: 'MyProfile' } },
     { properties: { title: 'Workers' } },
     { properties: { title: 'Clockings' } },
     { properties: { title: 'DrawerMoney' } },
     { properties: { title: 'Notifications' } },
     { properties: { title: 'LeaveRequests' } },
     { properties: { title: 'Shifts' } },
     { properties: { title: 'AuditLog' } }
   ]}
   Store returned spreadsheetId as sheetId.

3. Move spreadsheet into folder: PATCH https://www.googleapis.com/drive/v3/files/{sheetId}?addParents={folderId}&removeParents=root

4. Write header rows to each sheet using batchUpdate:
   - Workers: WorkerID, Name, Email, Phone, PayPerHour, Designation, JoinDate, Status
   - Clockings: WorkerID, Name, Date, ClockIn, ClockOut, BreakMins, HoursWorked, DayTotal
   - DrawerMoney: WorkerID, Name, Date, DrawerAmount, Notes, PhotoURL
   - Notifications: NotifID, Type, ToWorkerID, FromUID, Title, Body, Timestamp, Read, Status
   - LeaveRequests: RequestID, WorkerID, Name, StartDate, EndDate, Reason, Type, CoveringWorker, Status, ReviewedBy, ReviewedAt
   - Shifts: ShiftID, WorkerID, Name, Date, StartTime, EndTime, AssignedBy
   - AuditLog: LogID, Timestamp, UserID, UserName, Action, TargetSheet, RowID, OldValue, NewValue

5. Set sharing permissions:
   - Admin (current user email): role 'writer', type 'user' via Drive Permissions API.
   - Default sheet is private. Manager and Worker access granted later by Admin.

6. Generate join link base:
   - PATCH Drive file permissions to add a 'reader' link share: { role: 'reader', type: 'anyone', withLink: true }
   - GET the sharing URL from Drive API response.
   - Construct deep link: workforce://join?sheetId={sheetId}&token={sharingToken}&companyId={uid}
   - Store link in Redux state.user.joinLink and write to MyProfile sheet row.

7. Return { sheetId, folderId, joinLink }.

IMPORTANT: Wrap entire function in try/catch. On any step failure, log to console and rethrow with a descriptive message. The caller (NewUserProfileModal) will catch and show an error alert with a "Retry" button.

  function calcMonthlyPay(clockingRows, payPerHour) {
    const totalHours = clockingRows.reduce((sum, row) => sum + parseFloat(row.HoursWorked || 0), 0);
    return Math.round((totalHours * payPerHour) * 100) / 100;
  }

     function haversineDistance(lat1, lon1, lat2, lon2) {
       const R = 6371000;
       const dLat = (lat2 - lat1) * Math.PI / 180;
       const dLon = (lon2 - lon1) * Math.PI / 180;
       const a = Math.sin(dLat/2)**2 + Math.cos(lat1 * Math.PI/180) * Math.cos(lat2 * Math.PI/180) * Math.sin(dLon/2)**2;
       return R * 2 * Math.atan2(Math.sqrt(a), Math.sqrt(1-a));
     }
